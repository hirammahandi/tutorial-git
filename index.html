Ejemplo de trabajo con Git: Autobiografia

Para configurar el "username" y el "email" que va detectar Git se usa los comandos git --global user.name "nombre de usuario" y git --global user.email "email".

Para listar todos los usuarios y emails que estan configurados en en git se usa el comando git config --get-all key[value]. ejemplo:git config --get-all user.name.

Para inicializar git en un proyecto se usa el comando git init y es entonces q internamente git crea dos espacios en memoria llamados "staging" y "repo", donde "repo" es una carpeta oculta q se llama .git e inicializa una rama "master".

Para almacenar los archivos q van a ser versionados y/o actualizados y almacenados se usa el comando git add [nombre del archiv] o git add . para agregar todos los archivos dentro de la carpeta del proyecto al espacio "staging". Y para quitarlos del almacen (del staging) se usa el comando git rm --cached [file]

Para restaurar los cambios hechos en cualkier fichero del proyecto se usa el comando git restore [file]

Para pasar los archivos del proyecto al "repo" se usa el comando git commit especificando un mensaje descriptivo de lo que se va a subir por tando el comnado seria git commit -m "cualquier mensaje"

Git trabaja con ramas y la primera rama q crea es "master" donde se hace el primer commit. Es la rama donde se guarda la version mas actual y funcional del proyecto el cual se llama "head".

Para ver el estado actual del proyecto se usa el comando git status que permite saber si los o algun archivo del proyecto se encuentra en el estado "ontrack" u "track"

Para ver los commit q se han hecho se usa el comando git log --oneline mostrando los commit de la siguiente manera id [(HEAD -> master) solo si el commit es el head] mensaje del commit.
