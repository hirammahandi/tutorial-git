Ejemplo de trabajo con Git: Autobiografia

Para configurar el "username" y el "email" que va detectar Git se usa los comandos git --global user.name "nombre de usuario" y git --global user.email "email".

Para listar todos los usuarios y emails que estan configurados en en git se usa el comando git config --get-all key[value]. ejemplo:git config --get-all user.name.

Para inicializar git en un proyecto se usa el comando git init y es entonces q internamente git crea dos espacios en memoria llamados "staging" y "repo", donde "repo" es una carpeta oculta q se llama .git e inicializa una rama "master".

Para almacenar los archivos q van a ser versionados y/o actualizados y almacenados se usa el comando git add [nombre del archiv] o git add . para agregar todos los archivos dentro de la carpeta del proyecto al espacio "staging". Y para eliminarlo del almacen (del staging) se usa el comando git rm --cached [file] Y para sacarlo del staging se usa el comando git restore --staged [file]

Para restaurar los cambios hechos en cualkier fichero del proyecto se usa el comando git restore [file].

Para pasar los archivos del proyecto al "repo" se usa el comando git commit especificando un mensaje descriptivo de lo que se va a subir por tando el comnado seria git commit -m "cualquier mensaje"

Git trabaja con ramas y la primera rama q crea es "master" donde se hace el primer commit. Es la rama donde se guarda la version mas actual y funcional del proyecto el cual se llama "head".

Para ver el estado actual del proyecto se usa el comando git status que permite saber si los o algun archivo del proyecto se encuentra en el estado "ontrack" u "track"

Para ver los commit q se han hecho se usa el comando git log --oneline mostrando los commit de la siguiente manera id [(HEAD -> master) solo si el commit es el head] mensaje del commit. Con el id del commit me permite acceder a ese commit y volver a esa version de lo q hice en el proyecto, es como navegar por cada commit, y para ello se usa el comando git checkout [id del commit]. Si se quiere volver a la rama mas actualizada con el commit mas reciente se usa el comando git switch - y si lo que se quiere es crear una subrama sobre el commit al que navegue se usa el comando git switch -c [nombre de la nueva subrama] para almacenar todos los cambios q se haya hecho sobre ese commit al q se navego sin afectar a los demas.

Para ver un log de los cambios realizados sobre los archivos del proyecto se usa el comando git show

Cuando hacemos un commit con errores tenemos 2 opciones eliminarlos para siempre o crregirlos, y para evitar o resolver este tipo de cosas se usa el comando git reset [id del commit al q se quiere regresar] pero antes tenemos q saber los commit q se hayan echo usando el comando git log --oneline. Existen dos tipos de reset git reset [id del commit] --hard que lo q hace es eliminar el commit HEAD completo y el otro es git reset [id del commit] --soft que lo q hace es elminar el commit HEAD pero mantiene los cambios en staging

Para saber todas las ramas del proyecto se usa el comando git branch. Para crear una rama en git se usa el comando git branch [nombre de la rama]. Para navegar entre las ramas tambien se usa el comando git checkout [nombre de la rama] Y si quieres crear una rama nueva y automaticamente cambiarse a ella se usa el comando git checkout -b [nombre la rama]

Para unir los cambios de una rama a otra se usa el comando git merge [nombre de la rama], pero para ello hay q estar ubicado en la rama donde se va hacer el merge, por ejemplo quiero unir master con develop me tengo q parar en la rama master y aplicar el comando git merge [nombre de la rama]

Una forma de realizar un commit sin tener q haber agregado a los archivos antes con el git add .es usar el comando git commit -am "Mensaje del commit"q lo q hace es hacer el commit y automaticamente agregar los archivos a staging, pero esto solo funciona con los archivos q se le hayan echo git add . anteriormente

Para saber a que repositorio remoto se esta apuntando se usa el comnedo git remote -v

Para clonar un repo se usa el comando git clone [url o ssh del proyecto] [opcional:nombre de la carpeta donde se va a guardar el proyecto clonado]

Las ramas q se empiezan por origin/nombre de la rama son las ramas q se encuentran en el repositorio remoto

Para subir los commit al repositorio remoto se usa el comando git push [origin [nombre de la rama]] Para bajar el ultimo commit de una rama determinada se usa el comando git pull [origin [nombre de la rama]]

Para sincronizar una rama local y una rama remota a la de hacer un push se usa el comando git push -u origin [nombre de la rama], de esta manera no hay q especificar a q rama se esta haciendo el push

Para saber si hay tags en nuestro proyecto se usa el comando git tag, que lo q hace es mostrarte el nombre de las versiones

para crear un tag se usa el comando git tag -a [nombre de la version] -m [mensaje descriptivo] [id del commit para versionar]
